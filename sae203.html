Web Application with Database

SAÉ 2.03 · Semester 2 · Group of 3 · RT3 Competency

As a group of three, we designed and deployed a web application hosted on a Linux virtual machine to browse and manage a database provided by the university — my first real experience with collaborative web development.

What we did

  • Set up a Linux virtual machine and installed a web server along with a database management system (MariaDB)
  • Modelled the provided data schema and implemented CRUD operations (Create, Read, Update, Delete)
  • Built the web interface in HTML/CSS and connected the forms to the database
  • Worked as a team of three using Git/GitHub to version the code and split tasks
  • Wrote a Gantt chart to plan and track the project timeline
  • Produced an installation and deployment guide (2–5 pages) explaining our technical choices
  • Presented the project in 5 minutes with a live demo, followed by 10 minutes of Q&A

Key concepts

  • Web architecture — how a web server, a database and a client browser work together
  • CRUD — the four basic database operations every web app relies on
  • Collaborative Git workflow — branches, commits, and resolving merge conflicts
  • Gantt chart — planning and distributing tasks across a project timeline

Critical learning outcomes

AC0314

Web architecture — I understood how the different layers of a web application fit together: server, database, and client.

AC0315

Data management — choosing and configuring a DBMS, designing a schema that actually fits the business need.

AC0316

Collaborative work — using Git as a team for the first time was harder than expected, but it taught me how essential clear task splitting and branching really are.

R107/R209

Resources used — programming fundamentals, web technologies, data sources and processing.

Technologies

Personal reflection

This project taught me that working in a team is as much about organisation as it is about technical skills. Early Git conflicts cost us time, but we quickly understood the value of proper branching and clear task ownership. Presenting and defending our choices in front of the teachers was also a valuable exercise — it pushed me to really understand what I had built, not just build it.
